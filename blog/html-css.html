Difference between relative, absolute and fixed positioning

Static positioning

Static position is the default position for all the html elements. Basically it means they are laid out in the same position in the web browser as they are positioned in the html documents i.e., they stay where they are.

There are other element positioning such as relative, absolute and fixed.

Relative positioning

Relative positioning places an element in the webpage relative to normal static position of the html document. You can move the element a little bit, relative to its original place or the other nearby elements by using css tags like left, right, top, bottom or a combination of them by setting their position values. If one of the elements is applied with static positioning and the other with relative positioning, they can overlap. This is because the static position elements cannot be moved. Overlapping is not possible for static positioning, as the elements laid in the web browser are in the same position as they are in the html document.

As you can see from the image below, if you just use relative position without any css tags, the block remained in the same position as it is relative to the position of where it is supposed to be. You need to set the position and its value. Setting the top, right, bottom and left values will change the positioning of the block based on the values you have set to css tags.

Relative positioning using css tag top

As shown in the image below, applying css tag - "top" with value of 20px to the pink block moves it downward. This positions the pink block 20 pixels below its default top position. As a result, the pink block overlaps with the bottom purple block. This is the main disadvantage of relative positioning, if they are grouped nearby the positioning elements, they may overlap resulting in visual clutter.

Relative positioning using css tag right

When css tag - "right" with a value of 20px is applied, the pink block moves 20px to the right from the previous default position.

Relative positioning using css tag bottom

When css tag - "bottom" with a value of 20px is applied, the pink block moves 20px away from the bottom previous default position.

Relative positioning using css tag left

When css tag - "left" with a value of 20px is applied, the pink block moves 20px to the left from the previous default position.

Absolute positioning

There is not much movement for relative positioning, but absolute positioning is different. They can be moved anywhere on the website, as they are no longer in the flow and are not stuck in the same place. The element is positioned relative to the entire html document flow i.e., all the elements in the document.

Fixed positioning

Lastly, fixed positioning is they stay in one place. It is positioned relative to the view port of the website. If the element is in a fixed position, it wouldn't change even if you scroll the page i.e., the element stays in same position you fixed. Even when you change the viewport it still sticks to the place where we fixed it.
