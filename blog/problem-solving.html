Problem: Sometimes, I get stuck on what seem like silly typos or mistakes in my code. For example, I might see red squiggly lines under certain words, but I can't figure out what's wrong.

Lesson Learned: Always review my code for typos and syntax errors before diving deeper into potential issues.

Flip Flop Challenge

Problem: I was using an if/else statement to check if a number is divisible by 3, but I wasn't familiar with how to use the remainder operator effectively within the condition.

Techniques:

  1. Research: I looked into different resources like the handbook and Mozilla Developer Network (MDN) to understand the remainder operator (%). The documentation helped me break down the expression if (i % 3 === 0).

  2. Understanding Remainder: I learned that the remainder operator calculates the leftover value after dividing one operand by another. It also considers the sign of the dividend (the number being divided).

  3. Example Breakdown: The example if (i % 3 === 0) checks if the remainder of i divided by 3 is equal to 0. This indicates that i is evenly divisible by 3.

  4. Running Examples: I used the provided examples (console.log(13 % 5); etc.) to solidify my understanding of the remainder operator's behavior with positive, negative, and even numbers.

Feelings: As I explored the examples and documentation, I felt a sense of realization ("Oh, that's how it works!").

Lesson Learned: Breaking down complex code into smaller, testable parts can significantly improve comprehension. Additionally, referring to documentation and examples are crucial for understanding new operators and concepts.

Super FlipFlop Challenge

Problem: I needed to write code that iterates through a list of numbers and creates a new array containing strings like "Flip," "Flop," or "FlipFlop" based on divisibility rules. Numbers divisible by 3 would be mapped to "Flip," those divisible by 5 to "Flop," and those divisible by both 3 and 5 to "FlipFlop." The challenge lay in implementing the logic to add these corresponding strings to the new array.

Techniques:

  1. Understanding Loops: I reviewed the concept of for loops, which allow iterating through a list of elements and executing code for each element.

  2. Identifying the Right Method: I explored different ways to add elements to an array and realized the push() method was the most appropriate solution.

Solution with Push Method: I referred to my understanding of for loops and the push() method to write the code that iterates through the original array. Inside the loop, I used conditional statements to check if a number is divisible by 3, 5, or both. Depending on the divisibility, I used push() to add the corresponding string ("Flip," "Flop," or "FlipFlop") to the new array.

Feelings: Initially, I felt frustrated because I was struggling to integrate the push() method correctly. However, by reviewing loop concepts and analyzing error messages in the test code, I eventually figured out the syntax and logic. This led to a sense of accomplishment when the code worked as intended.

Lesson Learned: This challenge emphasized the importance of understanding both control flow structures (like for loops) and built-in array methods (like push()). Combining these concepts allows for efficient manipulation of data within arrays. Additionally, analyzing error messages provides valuable insights into syntax errors and logic issues.

Calculating Average in JavaScript

Problem: I was unsure how to calculate the average of an array of numbers.

Techniques:

  1. Breaking Down the Problem: I divided the task into smaller steps:

    Declare a variable to store the total sum.

    Iterate or go through each number in the list and add it to the total.

    Average is calculated by dividing the total sum by the array's length.

  2. Using a for Loop: I used a for loop to go through each number in the list.

Feelings: Visualizing the process of finding the average made it easier to write the code. Breaking it down into smaller steps helped me understand each part.

Lesson Learned: Breaking big problems into smaller parts makes them less scary and easier to solve. This helps me understand the logic and write better code.
